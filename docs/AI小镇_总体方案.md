# AI小镇总体方案（v2）

## 1. 产品目标

打造一个“高级美术 + 强交互 + 自主角色 + 世界演化”的单机模拟游戏：

- 不做固定脚本镇，而是可长期演化的社会系统
- 不做面板点击器，而是高反馈的沉浸式交互
- 不做重硬件限定，而是可分档运行并向高配释放画质与智能

对标方向：

- 研究层：Stanford Generative Agents / HKUST AI Town
- 体验层：高质感生活模拟 + 社会经营 + 动态叙事

---

## 2. 设计支柱（Pillars）

1. 角色可信：NPC有记忆、动机、关系、立场和成长
2. 世界可变：产业、建筑、文化、权力关系都会改变
3. 玩家有影响：每次互动都能改变局部，再汇总成全局变化
4. 视觉高级：统一美术语言、镜头叙事、可读性强的高级UI
5. 操作顺手：键鼠和手柄都能三步内完成核心操作

---

## 3. 可操作玩法系统（重点增强）

## 3.1 玩家可执行动作（Player Verbs）

- `观察`：查看人物状态、关系图、近期记忆、行为原因
- `对话`：闲聊、说服、谈判、调解、招募、追问
- `交易`：买卖、签长期订单、定价协商、合资投资
- `建设`：改造建筑、划分功能区、设公共设施
- `治理`：发起提案、投票、预算调配、税率与补贴
- `组织`：举办节庆、比赛、展览、夜市、论坛
- `调查`：追踪传闻、找证据、拆解冲突链
- `创作`：制作商品、发布内容、塑造个人品牌

## 3.2 操作映射（键鼠默认）

- `W/A/S/D`：移动
- `E`：主交互
- `Q`：快速观察（情绪/关系短窗）
- `Tab`：情报轮盘（人物/地点/事件）
- `1/2/3/4`：工具位（对话、交易、建设、治理）
- `M`：小镇全景图与热力图
- `J`：日志与任务线索
- `Space`：时间倍率切换（1x/2x/4x）

设计要求：

- 常用操作不超过两键组合
- 高频信息支持悬浮提示与快捷键回看
- 90%操作不依赖深层菜单

## 3.3 核心循环（分钟级）

1. 发现问题：在场景中看见冲突、机会或需求缺口
2. 选择干预：对话、交易、建设或政策介入
3. 立即反馈：情绪变化、价格变化、舆论变化可视化
4. 延迟反馈：日终出现关系变化、职业变化、事件连锁

## 3.4 可玩性增强模块

- 社会博弈：阵营、口碑、信任债务、利益交换
- 生活沉浸：天气、节日、昼夜、音乐、店铺作息
- 目标分层：短期赚钱、中期建区、长期改变小镇路线
- 变局机制：每周1个主题变局，逼迫玩家调整策略
- 探索奖励：隐藏人物线、秘密地点、稀有事件链

## 4. AI角色架构（自主思想 + 可控）

## 4.1 七段认知流水线

1. 感知：读取周边事件、对象关系、环境状态
2. 记忆检索：按相关度取近期与长期记忆
3. 动机计算：人格、需求、关系、风险偏好加权
4. 计划生成：输出可执行的行动序列
5. 执行动作：移动、交互、对话、交易
6. 反思总结：提炼本次事件的意义，更新长期倾向
7. 社会传播：信息进入传闻网络和社交圈

## 4.2 记忆体系

- 工作记忆：30-120分钟，负责连贯行为
- 情节记忆：7-30天，记录关系与冲突线
- 语义记忆：长期偏好、职业能力、价值观

记忆检索评分：

- `Score = 重要性 * 0.5 + 相关度 * 0.35 + 新近性 * 0.15`

## 4.3 行为引擎比例

- 规则与效用函数：65%
- 小模型对话与微计划：25%
- 导演系统关键节点：10%

说明：

- 这个比例既保证角色“像有思想”，也能保证稳定帧率和可控剧情密度。

---

## 5. 世界演化系统（随机但合理）

## 5.1 状态向量

- 经济：岗位、价格指数、产业集中度、现金流
- 社会：幸福度、治安、信任网络、阶层流动
- 空间：功能区活跃度、通行效率、租金压力
- 文化：节庆参与、创作热度、开放与保守倾向

## 5.2 导演系统规则

日终根据状态向量生成次日事件池：

- 机会事件：创业基金、招商、文化节、技术展
- 压力事件：供给不足、天气灾害、舆情危机
- 关系事件：合作、决裂、重组、迁入迁出

采样公式：

- `触发概率 = 基础概率 * 适配度 * 多样性系数 * 冷却系数`

---

## 6. 视觉与美术执行（高级化）

## 6.1 风格方向

推荐双路线并行评估，默认优先 `路线B`：

路线A：2.5D 手绘现实主义（Stylized Realism）

- 造型简洁，材质真实，色彩克制但有记忆点
- 光影重点放在“路径引导”和“情绪叙事”

路线B：Pixel RGB Premium（推荐）

- 星露谷式亲和像素语义 + 现代化动效反馈
- 对小团队产能和多平台性能更友好
- 更容易实现“高可读交互 + 高频内容更新”

对应详细规范见：`docs/交互美术与人物建模深化方案_v3.md`

## 6.2 画面分层

- 前景：动态植被、路人、粒子
- 中景：建筑、店招、角色主舞台
- 后景：天空盒、远景山体、城市轮廓

## 6.3 色彩与材质规范

- 主色：温暖木石中性色
- 点缀色：橙、青、红三类功能强调色
- 材质层级：木材、砖石、金属、玻璃、织物
- 夜景原则：局部高亮 + 大面积低照度，避免“整屏发光”

## 6.4 UI/UX原则

- 游戏世界优先，UI只提供决策信息
- 关键状态统一可解释：目标、情绪、关系、原因
- 每次系统变化都要有“看得见的反馈”
- 新手3分钟内能完成“观察-对话-干预-收获反馈”闭环

---

## 7. 技术架构与工程模块

## 7.1 技术栈

- 客户端：Godot 4
- 仿真：Python + FastAPI + asyncio
- 数据：SQLite + 向量检索插件（sqlite-vss 或 FAISS）
- 推理：llama.cpp 本地量化模型 + 可选云端兜底

## 7.2 模块划分

1. `client`：渲染、输入、镜头、UI、音频
2. `world-sim`：时间、天气、地块、经济更新
3. `agent-core`：需求、计划、记忆、关系图
4. `dialogue`：对话生成、摘要、语气控制
5. `director`：事件编排、密度控制、主题周
6. `telemetry`：日志、回放、解释面板、调参

## 7.3 调度策略

- 近景NPC：1-2秒一次决策
- 中景NPC：5-8秒一次决策
- 远景NPC：15-30秒一次聚合更新
- 日终批处理：反思、关系图更新、事件抽样

---

## 8. 硬件分档目标

## 8.1 基础档（兼容）

- 设备：16GB内存，无独显或入门显卡
- 目标：1080p / 45 FPS / 高精仿真NPC 20-30

## 8.2 高配档（i9 + RTX 4060）

- 目标：1080p 高画质 60 FPS 或 1440p 中高画质 50+ FPS
- 高精仿真NPC：40-60
- 常驻NPC：150-220
- 本地模型：7B/8B Q4-Q5 实时对话可用

详细参数见：`docs/高配笔记本_性能与画质档位.md`

---

## 9. 可测试质量门槛

- 连续30天仿真无经济死锁
- 每20分钟至少触发1次“意外但合理”事件
- NPC行为解释覆盖率 >= 90%
- 新手前10分钟留存目标 >= 80%
- 高配档平均帧率 >= 60（1080p 高画质）

---

## 10. 当前建议范围

首发版本控制在：

- 1张中型地图（镇中心/住宅区/商业区/工业边缘）
- 30个高精仿真NPC + 80个低频背景NPC
- 4类系统事件（经济、关系、文化、治理）
- 2条主线 + 6条角色支线 + 12个日常微事件模板

执行节奏见：`docs/MVP执行清单.md`
