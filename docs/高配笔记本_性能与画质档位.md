# 高配笔记本性能与画质档位（i9 + RTX 4060）

适用场景：

- CPU：Intel i9（移动端或桌面端）
- GPU：RTX 4060（常见8GB显存）
- 内存：建议32GB（16GB也可运行）

目标：

- 让你在“可玩 + 好看 + AI不弱智”之间拿到更高上限，而不是只跑低配模式。

---

## 1. 推荐运行档位

## 档位A：高画质优先（推荐首选）

- 分辨率：1920x1080
- 帧率目标：60 FPS
- 画质：高
- 高精仿真NPC：40
- 常驻NPC：150
- 本地对话模型：7B/8B Q4

适合：边玩边调试，整体观感和稳定性平衡最好。

## 档位B：仿真密度优先

- 分辨率：1920x1080
- 帧率目标：50-60 FPS
- 画质：中高
- 高精仿真NPC：60
- 常驻NPC：220
- 本地对话模型：7B/8B Q4，关键节点可临时升Q5

适合：压测AI行为与事件密度。

## 档位C：展示观感优先

- 分辨率：2560x1440
- 帧率目标：45-55 FPS
- 画质：高（关闭重度后处理）
- 高精仿真NPC：35-40
- 常驻NPC：120-150

适合：录制演示视频与外部汇报。

---

## 2. 图形设置建议（Godot 4）

建议开启：

- TAA 或 FSR2 Quality
- SDFGI（中等更新频率）
- 屏幕空间反射（中）
- 体积雾（低到中）
- 动态阴影（关键光源开启）

建议限制：

- 粒子总量做预算（避免夜市和节庆同时爆发时掉帧）
- 半透明叠加层数量限制
- 后处理中 Bloom 和 DOF 强度保持克制

---

## 3. AI推理配置建议

## 本地模型建议

- 首选：7B/8B 指令模型，Q4_K_M 或 Q5_K_M
- 兜底：3B模型用于低延迟设备
- 可选：13B Q4 只用于离线反思批处理，不用于实时对话

## 推理策略

- 实时对话：温度低、长度短、首句快返
- 日终反思：批处理，允许较长上下文
- 关键剧情：可切云端模型或高参数本地模型

---

## 4. 性能预算（建议上限）

- 客户端渲染：`<= 8ms`
- 仿真更新：`<= 5ms`
- 对话与AI：`<= 3ms`（不含异步推理等待）
- 其余系统：`<= 2ms`

总预算目标：16.6ms（60 FPS）

---

## 5. 参数模板（可直接放配置文件）

```ini
[render]
resolution = 1920x1080
fps_cap = 60
lod_near_radius = 22
lod_mid_radius = 48
shadow_quality = high
ssr = medium
volumetric_fog = medium
taa = true
fsr2 = quality

[simulation]
npc_total = 150
npc_high_fidelity = 40
npc_tick_near_sec = 1.2
npc_tick_mid_sec = 5.0
npc_tick_far_sec = 18.0
daily_batch_hour = 03:00

[ai]
model = 8b-q4
dialogue_max_tokens = 120
reflection_max_tokens = 320
dialogue_timeout_ms = 1800
fallback_template = true
```

---

## 6. 监控指标（开发期必须看）

- `FrameTime P95`：是否稳定在20ms以内
- `NPC Tick Delay`：高峰时决策延迟是否堆积
- `Dialogue Latency P95`：建议小于1.8秒
- `Event Diversity Index`：避免3天内事件高度重复
- `Memory Hit Rate`：对话是否正确召回相关记忆

---

## 7. 调优顺序（性能不够时）

1. 先降远景NPC更新频率
2. 再降后处理与体积雾
3. 再降高精仿真NPC上限
4. 最后才降分辨率

原则：

- 先保“玩法和智能”，再牺牲细节画质。
